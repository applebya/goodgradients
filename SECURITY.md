# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Good Gradients, please report it responsibly.

**Please do NOT open a public issue for security vulnerabilities.**

Instead, please report security issues by emailing the maintainers directly or using GitHub's private vulnerability reporting feature.

## Scope

Good Gradients is a client-side web application. Security considerations include:

- Cross-site scripting (XSS) in user-provided content
- Dependency vulnerabilities
- Local storage data handling

## What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

We aim to:

- Acknowledge receipt within 48 hours
- Provide an initial assessment within 7 days
- Release a fix as soon as possible for confirmed vulnerabilities

Thank you for helping keep Good Gradients secure!
